# Contributing

Danke, dass du zum Responsible Design Kit beitragen möchtest.

## Teams
- **Skills/Tool** — `.claude/skills/` und `packages/rdk/`
- **Landingpage** — `site/`
- **Testen** — `tests/`
- **Dokumentation** — `documentation/`

## Workflow
1. Issue anlegen oder zuweisen lassen
2. Feature-Branch von `main`
3. Änderungen committen (klare, kurze Commit-Messages)
4. Pull Request öffnen, Review anfordern
5. Nach Approval mergen

## Code-Style
- Markdown: 80–100 Zeichen Zeilenlänge, ATX-Headings (`#`)
- HTML: semantisch, barrierefrei
- JS: keine ungenutzten Imports, klare Namen

## Accessibility-First
Jede Änderung muss WCAG 2.2 AA berücksichtigen. Bei Unsicherheit: `.claude/skills/accessibility/SKILL.md` lesen.
